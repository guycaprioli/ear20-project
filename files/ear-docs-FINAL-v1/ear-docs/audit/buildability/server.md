# Buildability Assessment — ear-server Level 1 (M3–M4, WP-S1..S10)

**Question:** Can a Claude agent team (Fable lead, Opus contract-logic, Sonnet plumbing) build ear-server Level 1 into a working **DEV** stack (MinIO for R2, `replay` capture-source for HackRF, `S3_ENDPOINT_URL` the only switch) from the docs + runsheet (S11–S18) + bootstrap ALONE?

**Bar:** dev-env, not production. Real-R2 / real-HackRF / field-truth / legal gaps do NOT count as blockers here.

**Verdict (short):** **Buildable-with-marked-stubs, after two cheap doc-fixes and two ASK-gate answers.** The structure is exceptionally well-specified — models, ingest, per-agent DuckDB, pipeline job, handoff export are buildable as-written. But two whole-run assertions the runsheet's own gates require to go GREEN (conservation R5.1, byte-identity determinism) are provably wrong/unachievable as written; an agent following the contract literally builds a stack that **halts on the first quarantined row** and a determinism test that **fails on first honest implementation**. Those are not stubbable — they are the exit gates for S13/S17. They must be doc-fixed first, and the audit already wrote the exact fix. Two ASK gates (Q-03 Pass-2 choice at S15, Q-05 stub bias) genuinely gate their sessions.

## 3. Dev-env verdict

**buildable-with-marked-stubs** — after **two cheap doc-fixes (G1 conservation identity, G2 logical-determinism) and pinning G3 MIN_OBSERVATIONS + G8 stage-order**, all of which the audit has already written the exact fix for, and none of which need hardware/field/legal.

- **What ships as working dev code with no further input:** S11 models/admin/compose, S12 sync+onboarding gate, S13 two-table ingest (once G1's identity is corrected), S14 pipeline job + lock + health sweep, S15 Pass-1 + Pass-2 **stub** (P5 xfail — sanctioned), S16 noise filter + events (once G3 pinned), S17 Parquet export (once G2 corrected). The whole dev loop stands up on MinIO + replay.
- **The two genuine ASK gates** (Q-03 at S15, Q-05 at S04/S15) do **not** block the dev build because the project's own seam discipline (P5-xfail, DEFAULT-and-flag) is the sanctioned way past them — the runsheet explicitly routes S15 through a swappable stub. They block *finalizing the algorithm*, not *standing up the stack*.
- **The real risk is G1 and G2:** an agent that builds *literally to contract* produces a stack that halts on the first quarantine (G1) and a determinism test that fails on first honest run (G2). These are contract bugs, not stub-able seams — the fixture/gate encodes the wrong answer as law. They are one-edit fixes now; after S03/S04 author the fixture, they become fixture-re-derivation. **Fix G1/G2/G3/G8 before S03/S04 run.**

**Bottom line:** ~80% of the track is buildable-as-written to working dev code; the remaining ~20% is 4 cheap pre-session doc-fixes (G1/G2/G3/G8) + 2 sanctioned stubs (Pass-2, sweep-collapse) + 1 build-time refinement (fault-granularity). No hardware, cloud, or field truth needed to reach a green dev loop. A build session that starts S13/S16/S17 *without* the G1/G2/G3/G8 fixes would stall or cement wrong truth — so they are the load-bearing pre-work.
